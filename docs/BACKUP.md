# Резервное копирование и восстановление

Политика для продакшена: регулярные бэкапы, проверка восстановления, хранение копий вне сервера приложения.

## PostgreSQL

**Что критично:** база `audio_calls` (или ваша БД), все таблицы ORM: звонки, транскрипции, классификации, справочник тем, `pipeline_runs`, пользователи-менеджеры и справочники статусов.

**Снимок логически:**

```bash
pg_dump "$DATABASE_URL" -Fc -f "calls_$(date +%Y%m%d_%H%M).dump"
```

**Восстановление в пустую БД:**

```bash
pg_restore -d "$DATABASE_URL" --clean --if-exists calls_YYYYMMDD_HHMM.dump
```

Рекомендации:

- Автоматизировать ежедневный `pg_dump` (cron/Task Scheduler) и ротацию файлов.
- Хранить копии на отдельном диске/NAS или в объектном хранилище с контролем доступа.
- Периодически проверять восстановление на тестовый инстанс.

## Qdrant (векторный каталог тем)

Если включён `QDRANT_URL` и коллекция справочника синхронизирована из UI/скриптов:

- Снимайте **snapshot** коллекции средствами Qdrant или бэкапите том/директорию данных Qdrant по документации вашей установки.
- После массового изменения справочника в UI имеет смысл сделать точку восстановления.

Без Qdrant классификация по эмбеддингам может быть недоступна; тексты в PostgreSQL остаются.

## Redis

Используется как брокер Celery. **Персистентные бизнес-данные в Redis не хранятся** (очереди и результаты задач временны). Отдельный бэкап Redis для этого приложения обычно не требуется; при потере Redis достаточно перезапустить сервис (активные задачи могут потребовать повторной постановки).

## Файлы на диске (аудио, модели)

- **Аудиоархив** путей из пайплайнов — по политике вашей организации (NAS, репликация).
- **Модели** (Whisper, Gemma, эмбеддинги) — при необходимости держать копию установочных образов или кэша Hugging Face; восстановление по `requirements.txt` и переменным окружения.

## Шифрование и доступ

- Файлы бэкапов БД содержат персональные и коммерчески чувствительные данные — хранить шифрованными и с ограниченным доступом.
- Учётные данные к БД и Qdrant не помещать в репозиторий; использовать секреты CI/CD и `.env` только на сервере.
